Jaco Image Server

Production-ready self-hosted S3-compatible image storage. MinIO + Nginx reverse proxy with caching, CORS security, and public/private bucket support.

✓ Production Ready ⚡ Self-Hosted 🔒 Hardened Security

Architecture Overview

Browser / Mobile App
        │ HTTPS
        ▼
   Nginx (public :443)
        │
        ├── /media/public-bucket/*    ──► MinIO :9000  (cached, CORS)
        │
        ├── /media/private-bucket/*   ──► MinIO :9000  (signed URL, no cache)
        │
        └── /*  ─────────────────────────► Your App (Laravel, etc)

MinIO bound to 127.0.0.1:9000 (API) and 127.0.0.1:9001 (console) — no public exposure

Key Features

🪣 Dual Bucket System

Public bucket with 30-day Nginx cache and immutable Cache-Control. Private bucket requires signed URLs with zero cache.

🛡️ Security Hardened

Path traversal protection, executable extension blocking, CORS headers stripped and re-controlled, security headers (HSTS, nosniff, X-Frame).

⚡ Systemd Service

Hardened unit with NoNewPrivileges, ProtectSystem=strict, isolated filesystem.

🔄 Proxy Cache

Intelligent caching for public assets, cache invalidation for private objects, stale content during errors.

🚀 Easy Deployment

Automated installer for Debian/Ubuntu, one-command bucket creation, integration guides for Laravel.

📊 No Infrastructure Bloat

Runs on the same VPS as your main application. MinIO CE is lightweight—only accessible via localhost.

Quick Start

1

Clone & Install

git clone https://github.com/YOUR_ORG/jaco-image-server.git
cd jaco-image-server
sudo bash scripts/install.sh

The script creates a dedicated minio-user, downloads the MinIO binary, installs systemd service, and sets up Nginx cache directory.

2

Configure Credentials

sudo nano /etc/default/minio

Edit these two lines:

MINIO_ROOT_USER=your_admin_username
MINIO_ROOT_PASSWORD=your_very_long_random_password_32_chars_minimum

💡 Tip: Use openssl rand -base64 32 for secure passwords

3

Start MinIO Service

sudo systemctl start minio
sudo systemctl status minio

Service should show as active (running)

4

Create Buckets & App Account

sudo PUBLIC_BUCKET=myapp-public \
     PRIVATE_BUCKET=myapp-private \
     APP_ACCESS_KEY=myapp_app \
     bash scripts/create-buckets.sh

You'll be prompted for root password and app secret key. Save the credentials.

5

Configure Nginx

sudo cp nginx/cache.conf /etc/nginx/conf.d/minio-cache.conf
sudo cp nginx/cors-map.conf /etc/nginx/conf.d/minio-cors.conf
sudo nano /etc/nginx/conf.d/minio-cors.conf # Edit domain patterns

Add proxy locations to your server block or use include directive.

6

Test & Apply

sudo nginx -t
sudo systemctl reload nginx

Should return test successful

Laravel Integration

1. Add to .env

MINIO_ACCESS_KEY=myapp_app
MINIO_SECRET_KEY=your_app_secret_key
MINIO_ENDPOINT=http://127.0.0.1:9000
MINIO_PUBLIC_BUCKET=myapp-public
MINIO_PRIVATE_BUCKET=myapp-private
MINIO_PUBLIC_URL=https://yourdomain.com/media/myapp-public

2. Configure config/filesystems.php

'minio_public' => [
    'driver' => 's3',
    'key' => env('MINIO_ACCESS_KEY'),
    'secret' => env('MINIO_SECRET_KEY'),
    'region' => 'us-east-1',
    'bucket' => env('MINIO_PUBLIC_BUCKET'),
    'endpoint' => env('MINIO_ENDPOINT'),
    'use_path_style_endpoint' => true,
    'url' => env('MINIO_PUBLIC_URL'),
    'visibility' => 'public',
],

3. Use in Your App

// Public file
\Storage::disk('minio_public')->put('avatar.jpg', $stream);

// Private file (signed URL)
$url = \Storage::disk('minio_private')->temporaryUrl(
    'document.pdf', now()->addHours(1)
);

MinIO Console Access

The console is bound to 127.0.0.1:9001 for security. Use an SSH tunnel:

ssh -L 9001:127.0.0.1:9001 user@yourserver

Then open http://localhost:9001 in your browser.

Updating MinIO

sudo curl -fsSL https://dl.min.io/server/minio/release/linux-amd64/minio \
    -o /usr/local/bin/minio
sudo systemctl restart minio
sudo systemctl status minio

Check MinIO releases for changelog and breaking changes.

Contributing

We welcome issues and PRs! A few guidelines:

🚫 Never commit:

  • • Real credentials or passwords
  • • Private keys, certs, or .pem files
