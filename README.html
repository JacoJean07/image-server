Jaco Image Server

Production-ready self-hosted S3-compatible image storage. MinIO + Nginx reverse proxy with caching, CORS security, and public/private bucket support.

✓ Production Ready ⚡ Self-Hosted 🔒 Hardened Security

Architecture Overview

Browser / Mobile App
        │ HTTPS

        │

MinIO bound to 127.0.0.1:9000 (API) and 127.0.0.1:9001 (console) — no public exposure

Key Features

🪣 Dual Bucket System

Public bucket with 30-day Nginx cache and immutable Cache-Control. Private bucket requires signed URLs with zero cache.

🛡️ Security Hardened

Path traversal protection, executable extension blocking, CORS headers stripped and re-controlled, security headers (HSTS, nosniff, X-Frame).

⚡ Systemd Service

Hardened unit with NoNewPrivileges, ProtectSystem=strict, isolated filesystem.

🔄 Proxy Cache

Intelligent caching for public assets, cache invalidation for private objects, stale content during errors.

🚀 Easy Deployment

Automated installer for Debian/Ubuntu, one-command bucket creation, integration guides for Laravel.

📊 No Infrastructure Bloat

Runs on the same VPS as your main application. MinIO CE is lightweight—only accessible via localhost.

Quick Start

1

Clone & Install

git clone https://github.com/YOUR_ORG/jaco-image-server.git
cd jaco-image-server
sudo bash scripts/install.sh
2

Configure Credentials

sudo nano /etc/default/minio
MINIO_ROOT_USER=your_admin_username
MINIO_ROOT_PASSWORD=your_very_long_random_password
3

Start MinIO Service

sudo systemctl start minio
sudo systemctl status minio
4

Create Buckets & App Account

sudo PUBLIC_BUCKET=myapp-public \
     PRIVATE_BUCKET=myapp-private \
     APP_ACCESS_KEY=myapp_app \
     bash scripts/create-buckets.sh
5

Configure Nginx

sudo cp nginx/cache.conf /etc/nginx/conf.d/minio-cache.conf
sudo cp nginx/cors-map.conf /etc/nginx/conf.d/minio-cors.conf
sudo cp nginx/minio-proxy.conf /etc/nginx/conf.d/
6

Test & Apply

sudo nginx -t && sudo systemctl reload nginx

Security Enhancements

✓ Enhanced Nginx Signature Validation

Improved validation for private bucket access. Update line 109 in nginx/minio-proxy.conf:

# Old (basic, but functional)
if ($arg_X-Amz-Signature = "") {
    return 403 "Access denied";
}

# New (more robust)
if ($args !~* "X-Amz-Signature=.+") {
    return 403 "Access denied";
}

💡 This provides stronger validation of the signature parameter presence and format.

⚡ Add Rate Limiting (Optional)

Protect against brute force and DDoS attacks. Add to your Nginx server block:

# Define rate limit zones (at http{} level)
limit_req_zone $binary_remote_addr zone=minio_public_limit:10m rate=100r/s;
limit_req_zone $binary_remote_addr zone=minio_private_limit:10m rate=50r/s;

# Apply to public bucket location
location /media/YOUR_PUBLIC_BUCKET/ {
    limit_req zone=minio_public_limit burst=200 nodelay;
    ...(existing config)
}

# Apply to private bucket location
location /media/YOUR_PRIVATE_BUCKET/ {
    limit_req zone=minio_private_limit burst=100 nodelay;
    ...(existing config)
}

💡 Adjust rates based on your traffic patterns. Monitor with: tail -f /var/log/nginx/error.log | grep "limiting requests"

📊 Enable MinIO Audit Logging (GDPR/Compliance)

For compliance requirements (GDPR, HIPAA, etc.), enable audit logging in /etc/default/minio:

# Enable audit webhooks to log all API calls
MINIO_AUDIT_WEBHOOK_ENABLE_auth=on
MINIO_AUDIT_WEBHOOK_URL_auth=http://127.0.0.1:5000/audit

# Or log to syslog (simpler option)
MINIO_AUDIT_WEBHOOK_ENABLE=on
MINIO_AUDIT_WEBHOOK_URL=syslog://localhost:514

# Enable request/response logging
MINIO_LOGGER="json"

💡 Monitor logs with: tail -f /var/log/syslog | grep minio
